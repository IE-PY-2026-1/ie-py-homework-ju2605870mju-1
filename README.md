## 2. 시나리오 (5~10줄)
* (이곳에 한 학기 동안 개발할 프로그램의 전체 시나리오를 개괄적으로 요약합니다.)
* 이 프로그램은 기상청이나 환경공단 데이터를 시뮬레이션하여, 지역의 대기질(미세먼지, 오존) 데이터를 직접 기록하고 분석하는 콘솔 기반 관리 시스템입니다. 사용자가 오늘의 미세먼지(PM10)와 오존(O3) 농도를 실수형 데이터로 입력하는 것으로 프로그램이 시작됩니다. 입력된 수치를 바탕으로 환경부 기준에 맞춰 대기질 상태(좋음, 보통, 나쁨, 매우 나쁨)를 조건문으로 정밀하게 판별합니다. 판별된 등급에 따라 "KF94 마스크 착용 필수", "야외 활동 자제"등의 실생활 맞춤형 행동 가이드를 출력해 줍니다. 단순히 한 번 확인하고 끝나는 것이 아니라, 입력된 매일의 데이터는 리스트(배열)에 차곡차곡 누적되어 저장됩니다. 사용자는 메뉴판에서 번호를 선택해 새로운 수치를 입력하거나, 일주일 동안 가장 공기가 안 좋았던 날을 검색하는 등 다양한 기능을 계속해서 이용할 수 있습니다.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**

  * 이번 과제는 사용자가 입력한 대기질 측정값을 바탕으로 환경 등급을 진단하고 맞춤형 가이드를 제공하는 Air Guard(지능형 대기질 모니터링 시스템)을 구축하는 것이다.
  * input()함수를 통해 측정 지역명(str)과 미세먼지/오존 농도(float)을 반복 입력받아 빈 리스트 append로 누적 저장함.
  *리스트에 저장된 데이터를 기반으로 len()과 min(), max()함수를 사용하여 평균 및 최대값을 계산함.
  * if-elif-else문과  and/or 연산자를 결합하여 수치에 따라 3단계(좋음~나쁨) 대기질 등급을 판정하는 핵심 로직을 구현함.
  * 비정상 수치(음수 등) 입력 시 continue로 스킵하고, 기기 오류 수준의 수치 입력 시 break로 강제 종료하는 데이터 필터링 기능을 추가함.
    
### **🤖 AI 파트너십 과정**

1. **내용 1: 대기질 데이터 분석 로직 구현**
    * **프롬프트 요약:** "사용자가 입력한 미세먼지(PM10)와 오존(O3)데이터를 바탕으로 평균을 계산하고, 이를 기준으로 대기질 등급을 판단하는 프로그램을 어떻게 구현할 수 있을까?"
    * **적용 내용:** 사용자가 입력한 데이터를 리스트에 저장한 후 sum()과 len()내장함수를 활용하여 평균값을 걔산하였다. 계산된 평균 미세먼지와 오존 값을 기준으로 if-elif-else조건문을 사용하여 "좋음/보통/나쁨"등급을 판정하도록 구현하였다.
2. **내용 2: 비정상 데이터 처리 및 반복 구조 개선**
    * **프롬프트 요약:** "입력 데이터 중 오류 값(음수, 비정상적으로 큰 값)을 어떻게 처리하여 프로그램의 안정성을 높일 수 있을까?"
    * **적용 내용:** continue는 수치가 음수로 입력된 경우 대기질 수치가 음수일 수는 없으므로, 잘못된 입력으로 간주하고 리스트에 저장하지 않은 채 다음 측정으로 넘어                       갈 때 사용하고 break는 수치가 2000이상 등 현실적으로 불가능한 값이 입력된 경우, 이는 측정 센서 고장 등 오류로 간주되어 프로그램 자체를 즉시 멈출                      때 사용함.
    
### **🛠️ Troubleshooting & 기술 회고:**

1. **문제 1: 숫자를 입력했는데 계산이 안 되는 현상 (TypeError)**
    * **원인:** input()으로 입력받은 값이 문자열 상태라 계산이 불가능 했음.
    * **해결:** 데이터 입력 즉시 float()함수를 감싸 실수형으로 변환하는 형변환처리를 하여 정상적인 비교 및 산술 연산이 가능하도록 해결함.
2. **문제 2: 등급 판정 오류**
    * **원인:** 수치에 따라 등급을 나눌 때, '나쁨'이 나와야 할 수치가 '보통'으로 출력되는 현상이 있었다. if-elif문의 구간 범위를 겹치게 설정하여 논리적 구멍이 발생함 
    * **해결:** 미세먼지 기준표를 다시 확인하고, 관계 연산자(<,=,>)의 논리 연산자(and)를 사용해 각 elif 구간의 상한선과 하한선을 엄밀하게 재정의하여 정확한 등급
              판별을 구현함.
